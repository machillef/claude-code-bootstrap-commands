## User Stories

Numbered list. Every actor, every interaction, every benefit.

Format: `N. As a <actor>, I want <feature>, so that <benefit>`

Rules:
- Be exhaustive — cover all actors and all ways they interact with the system
- Be specific — "see account balance" not "manage account"
- The "so that" clause states a real user benefit, not a restatement of the feature
- One interaction per story — split compound stories ("and" in the feature = two stories)
- Include error/edge cases as stories where the user's experience matters (e.g., "As a user, I want a clear error when my file is too large, so that I know what to fix")
